1 Входные данные — текст задачи + решение

Это всё, что подаётся на вход. Обычный текстовый файл:

# файл: test_problem.txt Задание 18 Профильного ЕГЭ по математике. Найдите все значения параметра a, при каждом из которых уравнение 9^x − 4·3^x + a = 0 имеет ровно два решения. Решение: Замена t = 3^x, t > 0. Тогда t² − 4t + a = 0, то есть a = −t² + 4t. Обозначим f(t) = −t² + 4t = −(t − 2)² + 4, t > 0. Это парабола ветвями вниз с вершиной (2; 4). f(t) → −∞ при t → +∞, f(0) = 0 (не включаем, t > 0). Каждому t > 0 соответствует ровно одно x = log₃(t). Значит, число решений по x = число корней t > 0 уравнения f(t) = a. Горизонтальная прямая y = a: - a < 0: 1 пересечение (правая ветвь) → 1 решение - a = 0: 1 пересечение (при t = 4, т.к. t = 0 не входит) → 1 решение - 0 < a < 4: 2 пересечения → 2 решения ✓ - a = 4: 1 пересечение (вершина, касание) → 1 решение - a > 4: 0 пересечений → 0 решений Ответ: 0 < a < 4.
Человек пишет задачу + решение. Движок делает всё остальное.

SYSTEM_PROMPT (~450 слов) объясняет LLM:

  • Какие типы объектов доступны (function_curve, hline, filled_between, ...)
  • JSON-формат ответа (canvas + steps)
  • Что expression — это Python-синтаксис (x**2, не x²)
  • Графический метод: y=f(x), горизонтальные прямые y=a
  • Полный пример с задачей x² − 2|x| = a

LLM получает текст задачи и сама решает:

  • Какую функцию рисовать
  • Какой canvas (диапазоны осей)
  • Где критические значения a
  • Какие метки и подписи ставить
  • Какие области закрашивать
  • Какой текст писать на каждом шаге

# _postprocess_plan() # 1. Замена переменных: t → x (наш evaluator знает только x) "-(t - 2)**2 + 4" → "-(x - 2)**2 + 4" # 2. Дроби в JSON: -9/4 → -2.25 (JSON не поддерживает /) "y": -9/4 → "y": -2.25 # 3. Синтаксис: ^ → **, ln → log "x^2" → "x**2" # 4. Merge: текстовые шаги без объектов склеиваются с соседями

5 StepByStepParameter + render_parameter_svg код

Движок проходит по шагам плана и пошагово наращивает визуализацию:

# execute_task18_plan() for step in plan["steps"]: blocks.append({"type": "text", "content": step["text"]}) for obj in step["objects"]: sp.add(obj) # добавляет в сцену sp.snapshot() # рендерит текущее состояние → SVG blocks.append({"type": "visual", "svg": ...})

Каждый snapshot содержит все предыдущие объекты + новые:

  • SVG 1: только кривая
  • SVG 2: кривая + 2 горизонтальные прямые
  • SVG 3: + метки «2 решения», «1 решение»
  • SVG 4: + закрашенная область
  • SVG 5: + ответ

Рендерер рисует: координатную сетку, оси со стрелками, подписи делений, кривые как polyline (400 точек), горизонтальные прямые, заливки, метки с фоном.
