# Отчёт об улучшениях пайплайна геометрических визуализаций

**Дата:** 2026-05-06  
**Файлы:** `pipeline/geometry_renderer.py`, `pipeline/geometry_auto.py`

---

## Итерация 1 — Архитектурные улучшения

### D1. Актуализация SYSTEM_PROMPT
**Проблема:** LLM не знала о 60% возможностей рендерера → не использовала constraints.  
**Решение:** Добавлены в промпт:
- Constraints-солвер: `midpoint`, `median`, `altitude`, `bisector`, `circumscribed_circle`, `inscribed_circle`, `angle_arc`, `intersection`
- Новые примитивы: `chord`, `sphere`, `cube`, `parallelepiped`, `parallel_lines`, `equal_segments`, `bisector_line`
- Штриховка: `hatch` стили для polygon и cross_section
- Подписи длин: `length_label` на сегментах

**Результат:** LLM теперь использует `perpendicular`, `parallel_lines` и другие constraints вместо ручного вычисления координат.

---

### A2. Валидация планов + retry LLM
**Проблема:** ~30% планов содержали ссылки на несуществующие точки, невалидные constraints.  
**Решение:**
- `_validate_plan()` возвращает список warnings (проверяет points, segments, polygons, constraints)
- `_build_fix_request()` формирует сообщение с ошибками для LLM
- `plan_geometry_steps()` делает до 2 retry при наличии ошибок
- `_postprocess_plan()` распознаёт все типы constraints (не только `right_angle_marker`)

**Результат:** В тестах — 0 warnings с первого раза на обеих задачах (2D и 3D).

---

### C2. Intersection constraint
**Проблема:** LLM неправильно вычисляла координаты пересечения диагоналей/прямых.  
**Решение:** Новый constraint в solver:
```json
{"type": "intersection", "line1": ["A","B"], "line2": ["C","D"], "result_id": "P"}
```
Автоматическое вычисление через `_line_intersection()`.

**Результат:** Solver вычисляет точки пересечения точно; LLM не нужно угадывать координаты.

---

### E1. Авто-видимость рёбер в 3D
**Проблема:** LLM часто неправильно определяла, какие рёбра "за телом" (hidden).  
**Решение:**
- Новое поле `"faces"` в 3D-планах (перечень граней тела)
- `_compute_edge_visibility()` — алгоритм на основе нормалей граней + centroid-ориентации
- `_auto_mark_hidden_edges()` автоматически помечает скрытые рёбра перед рендером
- Робастность к ошибкам в порядке обхода вершин (автоориентация нормалей)

**Результат:** Для тестовой пирамиды — 3 ребра корректно помечены как hidden автоматически.

---

### B3. Анимация step-by-step
**Проблема:** Все SVG показывались одновременно — длинная страница без фокуса.  
**Решение:** `generate_html_animated()` — новый режим HTML:
- Один шаг на экране, навигация кнопками «Назад/Далее»
- Клавиши ←→ и пробел
- CSS fade-in анимация при переключении
- Прогресс-бар (точки)
- CLI флаг `--animated`

**Результат:** Компактный интерактивный формат, подходящий для встраивания в LMS.

## Итерация 2 — Качество рендеринга

### P1. Label collision avoidance
**Проблема:** Подписи `AF=13` и `h=60/13` рендерились на одних пиксельных координатах.  
**Решение:** `_resolve_label_collisions()` в `geometry_renderer.py`:
- Детектирует пары подписей ближе 40px по X и 16px по Y
- Раздвигает вертикально с минимальным шагом
- Два прохода для каскадных коллизий

**Результат:** Подписи гарантированно разнесены минимум на 16px по вертикали.

---

### P2. Визуализация ответа (не только текст)
**Проблема:** Ответ (высота, расстояние) показывался только как текстовая подпись, без отрезка на чертеже.  
**Решение:** Добавлен блок «визуализация ответа» в SYSTEM_PROMPT:
- Если ответ — длина → нарисовать отрезок (segment / perpendicular)
- Если ответ — угол → angle_arc
- Если ответ — площадь → hatch polygon

**Результат:** LLM использовала `perpendicular` constraint для высоты BH в тестовой задаче.

---

### P3. Авторасчёт canvas 2D
**Проблема:** Hardcoded fallback `x:[-1,14], y:[-2,8]` не подходил для многих конфигураций точек.  
**Решение:** `_ensure_canvas_2d()`:
- Если LLM не дала canvas → вычислить из точек + 20% padding
- Если LLM canvas обрезает точки (clipped) → пересчитать
- Если LLM canvas в 5+ раз больше нужного (oversized) → пересчитать
- Если LLM canvas адекватный → оставить как есть

**Результат:** Для точек трапеции (1,0)–(10,5): auto-canvas `[-0.8, 11.8] × [-1.0, 6.0]` вместо `[-1, 14] × [-2, 8]`.

---

### Бонус: Санитизация JSON
**Проблема:** LLM иногда генерирует невалидный JSON (`"y":![-1,0]`, trailing commas).  
**Решение:** `_sanitize_json_values()` — препроцессор перед парсингом:
- Замена `![]` конструкций на `0`
- Удаление trailing commas
- Обработка NaN/Infinity
