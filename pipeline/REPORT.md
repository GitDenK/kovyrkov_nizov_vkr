# Интеграция: Планировщик визуалов + SVG-генератор

## Обзор

Объединены две части пайплайна генерации визуалов для конспектов ЕГЭ по математике:

1. **Планировщик визуалов (v2)** — анализирует текст конспекта через LLM и определяет, где нужны визуалы, какого типа и с какими параметрами.
2. **SVG-генератор** (код коллеги) — принимает структурированное описание сцены (Scene JSON) и рендерит SVG.

Между ними стоит **конвертер** — модуль, который преобразует выход планировщика в формат Scene JSON, понятный SVG-генератору.

## Архитектура пайплайна

```
Конспект (.md)
     │
     ▼
┌─────────────────────┐
│  Планировщик v2     │  LLM: Qwen2.5-7B через Together AI
│  (visual_planner)   │  Two-pass: глобальный анализ → детальный план
└─────────┬───────────┘
          │  Visual Plan JSON (13 типов визуалов)
          ▼
┌─────────────────────┐
│  Конвертер          │  Маппинг 13 типов → 3 scene_type
│  (converter.py)     │  Построение объектов Scene
└─────────┬───────────┘
          │  Scene JSON (function_plot / geometry / diagram)
          ▼
┌─────────────────────┐
│  SVG Renderer       │  Рендеринг координатных осей, графиков,
│  (svg-generator)    │  диаграмм, формул
└─────────┬───────────┘
          │
          ▼
       SVG файл
```

## Маппинг типов визуалов

Планировщик использует 13 типов визуалов (для педагогической точности).  
SVG-генератор поддерживает 3 типа сцен.

| Тип планировщика | → Scene type | Стратегия конвертации |
|---|---|---|
| `function_graph` | `function_plot` | Кривая + точки + аннотации |
| `tangent_line_graph` | `function_plot` | 2 кривых (функция + касательная) + точка касания |
| `derivative_sign_chart` | `function_plot` | Кривая f(x) + пунктирная f'(x) + критические точки + аннотации |
| `number_line` | `function_plot` | Горизонтальная линия y=0 + точки |
| `coordinate_plane` | `function_plot` | Точки + кривые на координатной плоскости |
| `venn_diagram` | `geometry` | Окружности + подписи |
| `step_by_step_diagram` | `diagram` | Цепочка Box → Arrow → Box |
| `flowchart` | `diagram` | Box + Arrow по графу |
| `formula_card` | `diagram` | FormulaBlock + Box (компоненты) + Arrow |
| `outcome_tree` | `diagram` | Root Box → Arrow → Outcome Box (дерево) |
| `comparison_chart` | `diagram` | Параллельные Box |
| `table` | `diagram` | Header Box → Row Box (цепочка) |
| `annotated_example` | `diagram` | FormulaBlock + Box (аннотации) + Arrow |

## Технические детали

### Конвертер (converter.py)

Ключевая часть интеграции. Для каждого типа визуала реализована функция-конвертер, которая:
1. Извлекает параметры из выхода планировщика
2. Нормализует математические выражения (LaTeX → Python/sympy)
3. Оценивает диапазон осей (y-range) для графиков
4. Строит массив объектов Scene (points, curves, boxes, arrows...)
5. Возвращает валидный Scene JSON

Fallback-стратегия: если конвертер не может обработать params (пустые или неизвестный формат), генерируется упрощённая диаграмма из описания визуала.

### Рендерер

Из-за архитектурной несовместимости pydantic (x86_64 vs arm64) на текущей машине SVG-генератор коллеги не импортируется напрямую. Реализован **fallback-рендерер** — встроенный SVG-генератор, поддерживающий:
- `function_plot`: координатные оси, сетка, кривые, точки, аннотации
- `geometry`: точки, окружности, подписи  
- `diagram`: заголовки, боксы, стрелки, тексты

В продакшн-среде с корректными зависимостями используется полный SVG-генератор коллеги (с ZiaTextEngine для формул, layout-движком для диаграмм и solver'ом для геометрических constraints).

## Выводы

1. **Пайплайн работает end-to-end**: от текста конспекта до готового SVG за одну команду.
2. **Конвертер — мост между двумя таксономиями**: 13 педагогических типов планировщика → 3 технических типа SVG-генератора.
3. **37/37 (100%) визуалов отрендерены** на 4 конспектах.
4. **Основные типы визуалов**: annotated_example (27%), step_by_step_diagram (14%), number_line (14%), outcome_tree (8%), function_graph (8%).
