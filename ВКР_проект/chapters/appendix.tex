\section*{Приложение Б. Список сокращений}
\addcontentsline{toc}{section}{Приложение Б. Список сокращений}

\begin{description}
    \setlength{\itemsep}{2pt}
    \item[API] \textit{(Application Programming Interface)} --- программный интерфейс приложения; набор соглашений о формате вызовов и обмене данными между программными компонентами.
    \item[ЕГЭ] Единый государственный экзамен.
    \item[JSON] \textit{(JavaScript Object Notation)} --- текстовый формат обмена структурированными данными на основе синтаксиса JavaScript.
    \item[LLM] \textit{(Large Language Model)} --- большая языковая модель; нейросетевая архитектура, обученная на текстовых корпусах для генерации и понимания текста.
    \item[RAG] \textit{(Retrieval-Augmented Generation)} --- генерация с извлечением информации; техника дополнения запроса к языковой модели релевантными фрагментами из корпуса.
    \item[SVG] \textit{(Scalable Vector Graphics)} --- формат масштабируемой векторной графики на основе XML, стандартизированный консорциумом W3C.
    \item[ФИПИ] Федеральный институт педагогических измерений.
    \item[HTML] \textit{(HyperText Markup Language)} --- язык гипертекстовой разметки; стандарт описания структуры веб-страниц.
    \item[URL] \textit{(Uniform Resource Locator)} --- унифицированный указатель ресурса; стандартный способ адресации ресурсов в сети Интернет.
\end{description}

\subsection*{В.3. Пример неудачной генерации изображения базовой линией}
% ───────────────────────────────────────────────────────────

На рис.~\ref{fig:app:badbaseline} приведён характерный пример выхода модели
text"=to"=image на геометрической задаче, используемый в качестве базовой
линии сравнения. Видно, что модель нарушает точные геометрические
отношения: описанная окружность
не касается вершин треугольника. Этот класс систематических дефектов
послужил одним из ключевых аргументов в пользу двухшагового подхода с
промежуточным JSON"=представлением.

\begin{figure}[htbp]
    \centering

    \caption{Пример выхода text"=to"=image на задаче «треугольник с медианой из угла B,
             вписанный в окружность»: описанная окружность не касается вершины~$A$.}

\subsection*{В.4. Полный пример работы конвейера на задаче №16}
\label{sec:app:task16-full}
% ───────────────────────────────────────────────────────────

В основном тексте на рис.~\ref{fig:ch3:trapezoid-steps} приведены четыре
ключевых кадра пошагового построения геометрической ветки конвейера на
задаче~16 ЕГЭ (трапеция с перпендикулярными диагоналями). На
рис.~\ref{fig:app:task16-full} показан полный конечный артефакт системы
для той же задачи: текст условия, четыре кадра и краткие пояснения к
каждому шагу. Все четыре кадра содержательно совпадают с подрисунками
рис.~\ref{fig:ch3:trapezoid-steps:s1}--\ref{fig:ch3:trapezoid-steps:s4}.

Соответствующие воспроизводимые артефакты собраны в каталоге
\texttt{examples/task16\_trapezoid\_repro/} репозитория проекта
(см.~приложение~А) и включают: формулировку задачи (\texttt{input/task16.md}),
сохранённый план визуализации, полученный от языковой модели
(\texttt{expected/auto\_task16\_plan.json}), HTML\nobreakdash-эталон с
четырьмя шагами (\texttt{expected/task16\_trapezoid\_4steps.html}) и два
сценария воспроизведения --- офлайн (без обращения к языковой модели,
\texttt{scripts/run\_offline.sh}) и онлайн (полный прогон геометрической
ветки через Together~AI, \texttt{scripts/run\_online.sh}). README в
корне каталога содержит подробные инструкции по сравнению полученного
выхода с эталоном.
